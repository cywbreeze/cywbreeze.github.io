java8新特性

发表于2022-11-02|更新于2022-11-02|Java
|阅读量:

总纲

  • 速度更快

  • 代码更少（增加了新的语法：==Lambda表达式==）

  • 强大的==Stream API==

  • Date Time API − 加强对日期与时间的处理

  • 便于并行

  • 最大化减少空指针的异常： Optional类

  • Nashorn 引擎，允许在JVM上运行JS应用

Ⅰ、Lambda 表达式

Lambda 是一个匿名函数，Lambda 允许把函数作为一个方法的参数（函数作为参数传递进方法中），我们可以把Lambda表达式理解为一段可以传递的代码（将代码像数据一样进行传递）。使用 Lambda 表达式可以使代码变的更加简洁紧凑。

1.举例：

  • Comparator com2 = (o1,o2) -> o1.compareTo(o2)

2.格式：

​ ==->== ： Lambda操作符 或 箭头操作符

​ ==->左边==：Lambda形参列表 （其实就是接口中抽象方法的形参列表）

​ ==->右边==： Lambda体 （其实就是重写抽象方法的方法体）

3.Lambda的使用（分六种情况）

  • ==总结==

    ==->左边==：Lambda 形参列表的参数类型可以省略； 如果Lambda参数列表只有一个参数，其一对 ( ) 可以省略；若Lambda 需要两个或以上的参数，多条执行语句，则不需要省略。

    ==->右边==：Lambda体使用一对{ }包裹；如果Lambda体只有一条语句，则可以省略 { } 以及 return 关键字。

4.Lambda表达式的本质：作为接口的实例

  • 语法格式一： 无参，无返回值

    • ```java
      public void lambdaTest(){
      //普通写法
      Runnable r1 = new Runnable() {
      @Override
      public void run {
      System.out.println(“hello world”);
      }
      };
      r1.run();

      //Lambda 表达式写法
      Runnable r2 = () -> System.out.println(“hello world”);
      r2.run();
      }

  • 语法格式三：数据类型可以省略，因为可由编译器推断得出，称为“==类型推断==”

    • ```java
      @Test
      public void lambdaTest2(){、
      Consumer con1 = (String s) -> {System.out.println(s);};
      con1.accept(“hello world”);
      //省略String
      Consumer con2 = (s) -> {System.out.println(s);};
      con2.accept(“hello world”);
      }

  • 语法格式五：Lambda 若只需要两个或以上的参数，多条执行语句，并且可以又返回值

    • ```java
      @Test
      public void lambdaTest4(){
      Comparator com1 = new Comparator() {
      @Override
      public int compare(Integer o1, Integer o2) {
      System.out.println(o1);
      System.out.println(o2);
      return o1.compareTo(o2);
      }
      };
      //Lambda 表达式如下
      Comparator com2 = (o1,o2) -> {
      System.out.println(o1);
      System.out.println(o2);
      return o1.compareTo(o2);
      };
      }

Ⅱ、函数式接口

  • 如果一个接口中只申明了一个抽象方法，则此接口称为函数式接口

  • 函数式接口可以被隐式转换为 lambda 表达式。

  • Lambda 表达式和方法引用（实际上也可认为是Lambda表达式）上。在接口上使用 @FunctionalInterface 注解，这样可以检查他是否是一个函数式接口

  • 如定义了一个函数式接口如下：

    • ```java
      @FunctionalInterface
      interface GreetingService
      {
      void sayMessage(String message);
      }

  1. java内置的核心函数式接口

    • 消费型接口 Consumer void accept（T t）

    • ```java
      @Test
      public void lambdaTest5(){
      happy(500, new Consumer() {
      @Override
      public void accept(Double aDouble) {
      System.out.println(“hello world:” + aDouble);
      }
      });
      //函数式接口如下
      happy(500,money -> System.out.println(“hello world:” + money));
      }
      public void happy(double money,Consumer con){
      con.accept(money);
      }

==情况二：类 :: 静态方法==

  • Comparator中的int compare(T t1, T t2)

  • Integer中的int compare(T t1, T t2)

  • ```java
    @Test
    public void lambdaTest6(){
    //Lambda表达式
    Comparator com1 = (t1,t2) -> Integer.compare(t1,t2);
    System.out.println(com1.compare(12,21));
    //方法引用
    Comparator com2 = Integer::compare;
    System.out.println(com2.compare(12,21));
    }

Ⅳ、Stream API

  • Java 8 API添加了一个新的抽象称为流Stream，可以让你以一种声明的方式处理数据。

  • Stream API 对集合数据进行操作，就类似与用 SQL 语句执行的数据库查询

  • Stream API可以极大提高Java程序员的生产力，让程序员写出高效率、干净、简洁的代码。

  • 这种风格将要处理的元素集合看作一种流， 流在管道中传输， 并且可以在管道的节点上进行处理， 比如筛选， 排序，聚合等。

  • stream 和 Collection集合的区别：Collection是一种静态的内存数据结构，而Stream是有关计算的，或者说前者面向内存的，后者面向CPU,通过CPU实现计算。

  • 集合讲的是数据，Stream讲的是计算！

  • ==注意==

    • Stream并不会存储元素，而是按需计算。
    • Stream不会改变源对象，相反，它们回返回一个持有结果的新Stream。
    • Stream操作是延迟执行的，这意味着他们会等到需要结果的时候才执行。

  • Stream 执行流程

    • Stream的实例化
    • 一系列的中间操作（过滤、映射…..）
    • 终止操作

  • 说明：

    • 一个中间操作链，对数据源进行处理
    • 一旦执行终止操作，就执行中间操作链，并产生结果，之后，不会再被使用

Stream 的中间操作

​ 多个中间操作可以连接起来形成一个流水线，除非流水线上触发终止操作，否者中间操作不会执行任何的处理！ 而在终止操作时一次性全部处理，称为“惰性求值”。

1-筛选与切片

forEach

Stream 提供了新的方法 ‘forEach’ 来迭代流中的每个数据。

1.filter(Predicate p) —接收Lambda，从流中排除某些元素

  • ```java
    Liststrings = Arrays.asList(“abc”, “”, “bc”, “efg”, “abcd”,””, “jkl”);
    // 获取空字符串的数量
    long count = strings.stream().filter(string -> string.isEmpty()).count();

3.limit(long maxSize)—截断流，使其元素不超过给定数量

  • ```java
    List strings = Arrays.asList(“abc”, “q”, “bc”, “efg”, “abcd”,””, “jkl”);
    // 获取前三个元素
    strings.stream().limit(3).foreach(System.out::println);

2-映射

map

map 方法用于映射每个元素到对应的结果，以下代码片段使用 map 输出了元素对应的平方数：

  • ```java
    List numbers = Arrays.asList(3, 2, 2, 3, 7, 3, 5);
    // 获取对应的平方数
    List squaresList = numbers.stream().
    map( i -> i*i).distinct().foreach(System.out::println);

Ⅴ、Optional类

  • Optional 类是一个可以为null的容器对象。如果值存在则isPresent()方法会返回true，调用get()方法会返回该对象。

  • Optional 是个容器：它可以保存类型T的值，或者仅仅保存null。Optional提供很多有用的方法，这样我们就不用显式进行空值检测。

  • Optional 类的引入很好的解决空指针异常。
